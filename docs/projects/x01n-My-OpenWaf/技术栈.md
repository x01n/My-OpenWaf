# My-OpenWaf 技术栈

> 项目：https://github.com/x01n/My-OpenWaf  
> 分析日期：2026-07-04

## 概述

My-OpenWaf 是一款自托管的高性能 Web 应用防火墙（WAF）/ 反向代理，采用**控制面 + 数据面**的双平面架构。后端基于 Go 语言，前端基于 Next.js 静态导出并嵌入后端管理界面。

## 基础设施与部署

| 项 | 说明 |
|----|------|
| **数据库** | SQLite（默认），支持切换 MySQL / PostgreSQL |
| **缓存** | Redis（可选，用于限流、挑战状态、配置同步） |
| **部署形态** | 自托管二进制 / Docker 容器 |
| **默认管理端口** | `:9443`（HTTPS） |
| **TLS 证书** | 内置 ACME 自动证书管理，支持 SNI 多证书 |
| **前端嵌入** | Next.js 构建产物（`out/`）复制到 `internal/core/adminweb/dist/` 由后端静态文件服务提供 |

## 技术选型原因推断

- **Hertz**：字节跳动开源的高性能 Go HTTP 框架，零拷贝、内置连接池，适合高并发 WAF 数据面处理。
- **GORM**：支持多数据库切换，便于从 SQLite 快速迁移到 MySQL/PostgreSQL 生产环境。
- **Next.js 静态导出**：前端不依赖服务端渲染，构建产物可直接嵌入 Go 二进制，简化部署。
- **Ristretto + Redis**：本地内存缓存（Ristretto）用于高频限流/反重放，Redis 用于分布式节点同步与持久化挑战状态。
- **go-captcha + WASM**：验证码与 5s 盾（WASM PoW 验证）结合，提供分层人机验证能力。
