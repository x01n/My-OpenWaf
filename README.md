# My-OpenWaf

**自托管的高性能 Web 应用防火墙（WAF）/ 反向代理，保护你的 Web 应用免受攻击与漏洞利用。**

---

## 👋 简介

My-OpenWaf 是一款自托管的 WAF（Web Application Firewall），通过反向代理架构在 Web 应用前端建立安全屏障，过滤、监控并拦截恶意 HTTP/S 流量，同时保障合法请求正常通行。

核心防护能力涵盖：

- **OWASP Top 10 攻击防御** — SQL 注入、XSS、命令注入、路径遍历、XXE、SSRF 等
- **ACL 访问控制** — 基于 IP / 地理位置 / 自定义规则的精细化访问控制
- **速率限制** — 防御暴力破解、DoS 攻击与流量滥用
- **机器人检测与人机验证** — 验证码挑战 + WASM 动态 5s 盾
- **IP 信誉系统** — 基于行为分析的 IP 信誉评分与自动封禁
- **多上游负载均衡** — 支持多后端节点负载均衡与健康检查
- **响应缓存** — 可配置的资源缓存策略，提升访问速度

---

## 🔥 核心功能

### 🛡️ Web 攻击防御

防御 OWASP Top 10 全品类 Web 攻击，包括 SQL 注入、XSS、代码注入、OS 命令注入、CRLF 注入、XXE、SSRF、路径遍历等。支持自定义规则引擎，按规则粒度配置处置动作。

### ⚡ 速率限制

抵御 DoS 攻击、暴力破解、流量激增等滥用行为，对超出阈值的请求执行限速处置（HTTP 429）。

### 🤖 人机验证

多种验证机制保护网站免受 Bot 攻击：

- **验证码挑战** — 交互式验证，区分人类与自动化程序
- **5s 盾（WASM 动态验证）** — 使用 Go/C++ 编译的动态 WASM 模块，内嵌 VMP 与自定义高效 VM，在浏览器环境中执行验证：
  - 检测无头浏览器 / 自动化框架（Puppeteer、Selenium 等）
  - 检测 DevTools 调试窗口
  - 所有验证逻辑在 VMP 内的 VM 执行，防止逆向破解
- **混合策略** — 支持验证码 + 验证码混合、验证码后限速等组合策略

### 🔒 访问控制

- **白名单** — 最高优先级，匹配即放行
- **黑名单** — 次优先级，匹配即执行处置动作
- **规则策略** — 基础规则引擎，支持灵活的条件组合

### 🌐 反向代理与负载均衡

- 多上游节点负载均衡
- TLS 终止与证书管理
- WebSocket 连接透传
- SSE 事件推送

### 📊 可观测性

- 安全事件实时记录与归档
- 访问日志写入 Redis 缓存
- 健康检查与性能指标收集
- 前端管理仪表板

## 🎯 策略动作体系

| 策略 | 状态码 | 说明 |
| --- | --- | --- |
| **限速** | `429` | 阻止高频访问资源或异常高频请求 |
| **拦截** | `403 / 418` | 响应拦截或触发验证码动作 |
| **阻断** | — | 直接返回 RST 并静默丢弃后续连接 |
| **人机验证** | `422` | 执行人机验证策略（验证码 / 5s 盾） |

**匹配优先级：** 白名单 > 黑名单 > 规则策略

## 🚀 快速开始

### Docker 部署

```bash
docker run -d \
  --name my-openwaf \
  -p 443:9443 \
  -v ./data:/app/data \
  my-openwaf:latest
```

### 从源码构建

**前置要求：** Go 1.25+、Node.js 22+、CGO 支持

```bash
# 构建后端
cd cmd && go build -o ../bin/my-openwaf .

# 构建前端
cd frontend && npm ci && npm run build
```
