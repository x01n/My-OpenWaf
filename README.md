# My-OpenWaf

动作:
 策略 - 状态码 - 操作
 限速 - 429 - 阻止高频访问资源或者异常高频访问
 拦截 - 403/418 - 响应拦截或者执行验证码动作
 阻断 - 无 - 直接对请求返回RST并静默丢弃后续
 人机验证 - 422 - 执行人机验证策略




 人机验证后续动作包括
 验证码与验证码的混合策略，验证码后限速的混合策略，支持自定义匹配策略

 白名单检查优先最高，其次匹配黑名单，然后是基础的规则与策略
 对于高危的资源耗尽owsap10攻击请求支持配置为阻断，支持按照规则来配置操作，比如规则xxxx配置为限速或者其他策略，支持多上游负载均衡，支持请求资源缓存提高速度（可以配置缓存的资源后缀或者路径等规则），请求日志与拦截日志写入redis缓存，支持多种人机验证规则，5s盾使用动态生成的wasm进行验证，wasm使用go或者cpp构建，验证的点包括浏览器环境（无头或者类自动化环境则不能通过，正常浏览器正常通过，包括调试窗口打开也是），wasm内部包含一个vmp与自定义的高效vm，全部的操作都要在vmp内的vm执行，防止遭到逆向破解
